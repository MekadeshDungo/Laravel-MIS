## Key Findings
- All blade views already exist in VET-MIS (moved to `Client/` subdirectory)
- Login and register views are **identical** between both projects
- `welcome.blade.php` is also **identical**
- The main difference is **14 image files** that exist in source but are missing from VET-MIS

#### 5c. Fix `kapon/form` route to use VET-MIS Owner model
Update the kapon/form route closure to use `Owner` instead of `PetOwner`:
```php
Route::get('/kapon/form', function () {
    $user = auth()->user();
    $owner = $user ? \App\Models\Owner::where('user_id', $user->id)->first() : null;
    $pets = $owner ? $owner->pets : collect([]);
    $petsArray = $pets->map(function($pet) {
        return [
            'id' => $pet->pet_id,
            'name' => $pet->pet_name,
            'species' => $pet->species,
            'breed' => $pet->breed,
            'age' => $pet->estimated_age,
            'weight' => $pet->pet_weight,
            'image' => $pet->pet_image
        ];
    })->toArray();
    return view('Client.kapon_form', compact('user', 'owner', 'petsArray'));
});
```
