# Veterinary Bite and Rabies Case Management System with Map Integration

## Overview
Create a unified "Exposure Case" module combining bite and rabies reports into a single case record that evolves from bite → suspected rabies → confirmed rabies. Only approved cases appear on the public map.

## User Preferences (Clarified)
- **Map Type**: Barangay Heatmap (show counts per barangay, not individual markers)
- **Coordinates**: Use barangay center points (auto-calculated from barangay data)
- **Existing Data**: Migrate all BiteRabiesReports to new ExposureCase records

## Existing System Analysis
The system already has:
- `BiteRabiesReport` model - stores bite/rabies reports from clinics, ABCs, hospitals
- `RabiesCase` model - linked to bite reports  
- Client submission form for bite reports
- Leaflet-based rabies geomap showing data by barangay (count-based, not individual cases)
- Notification system for bite reports
- Admin views for managing reports

## What's Missing (Gap Analysis)
1. **No unified "Exposure Case" model** - Currently BiteRabiesReport and RabiesCase are separate
2. **No geographic coordinates** - Reports store barangay_id but not lat/lng coordinates
3. **Current map shows count per barangay** - Not individual case markers
4. **No status-based visibility** - All reports show regardless of approval status
5. **No case evolution** - Can't update bite → suspected rabies → confirmed rabies in one record
6. **No public map** - Current geomap is admin-only

## Implementation Plan

### Phase 1: Database Schema & Migration

#### 1.1 Create ExposureCase Migration
Add new table `exposure_cases` to unify all bite/rabies cases:
- case_id (EXP-YYYY-XXXXX format)
- report_type: bite | suspected_rabies | confirmed_rabies
- status: pending | approved | closed
- Patient info (name, age, gender, contact, barangay)
- Animal info (species, status, owner, vaccination, condition)
- Bite details (date, site, category, nature)
- Report source (clinic, abc, hospital, facility name)
- Links to original BiteRabiesReport via original_report_id
- Uses barangay center lat/lng for map coordinates (from barangay.latitude/longitude)

- Already completed in previous task

### Phase 3: Admin Interface

#### 3.1 Create ExposureCaseController
```php
// Methods needed:
- index() - List all cases with filters (status, type, barangay, date range)
- pending() - Dashboard showing pending cases requiring approval
- show(ExposureCase $case) - View full case details
- approve(ExposureCase $case) - Change status to approved (publish to map)
- update(Request $request, ExposureCase $case) - Update case details including report_type
- close(ExposureCase $case) - Mark case as closed
- destroy(ExposureCase $case) - Delete case
```

#### 3.2 Create Admin Views
- `resources/views/exposure-cases/index.blade.php` - Table with filters, action buttons
- `resources/views/exposure-cases/pending.blade.php` - Pending cases dashboard
- `resources/views/exposure-cases/show.blade.php` - Case detail with edit capability
- `resources/views/exposure-cases/edit.blade.php` - Form to update report_type/status

#### 3.3 Add Routes
```php
// Admin routes
Route::middleware(['auth'])->prefix('admin')->group(function () {
    Route::get('/exposure-cases', [ExposureCaseController::class, 'index'])->name('exposure-cases.index');
    Route::get('/exposure-cases/pending', [ExposureCaseController::class, 'pending'])->name('exposure-cases.pending');
    Route::get('/exposure-cases/{case}', [ExposureCaseController::class, 'show'])->name('exposure-cases.show');
    Route::put('/exposure-cases/{case}/approve', [ExposureCaseController::class, 'approve'])->name('exposure-cases.approve');
    Route::put('/exposure-cases/{case}', [ExposureCaseController::class, 'update'])->name('exposure-cases.update');
    Route::put('/exposure-cases/{case}/close', [ExposureCaseController::class, 'close'])->name('exposure-cases.close');
    Route::delete('/exposure-cases/{case}', [ExposureCaseController::class, 'destroy'])->name('exposure-cases.destroy');
});
```

#### 3.4 Update Admin Navigation
- Add menu item in sidebar for "Exposure Cases"
- Show pending count badge
