# Meat Establishment Registration - Updated Plan

## User Request
Update the Meat Establishment Registration form to include more comprehensive fields.

## New Form Structure

### I. Establishment Info
- **Establishment Name** - Text Field
- **Establishment Type** - Dropdown with options:
  - Meat Shop/Meat Retailer
  - Slaughterhouse (Abattoir)
  - Livestock Farm
  - Poultry Farm
  - Meat Processing Plant
- **Business Permit No.** - Text Field

### II. Contact & Ownership
- **Owner Name** - Text Field
- **Contact Person** - Text Field
- **Contact Number** - Number Field
- **Email Address** - Email Field

### III. Location Details
- **Barangay** - Dropdown (existing)
- **Exact Address** - Text Area (Complete street address, building, or unit number)
- **Landmark (Optional)** - Text Field

## Implementation Steps

### 1. Database Changes
- Add `establishment_type` column to `meat_establishments` table (enum)
- Add `contact_person` column
- Add `contact_number` column
- Add `email` column
- Add `landmark` column
- Rename `permit_no` to `business_permit_no` (or just add)

### 2. Model Updates
- Add new fields to `$fillable`
- Add relationship if needed

### 3. Controller Updates
- Update `createEstablishment` method to pass required data
- Update `storeEstablishment` to validate new fields
- Update `updateEstablishment` to handle new fields

### 4. View Updates
- Completely redesign `meat_establishment_form.blade.php` with new sections
- Group fields by sections (Establishment Info, Contact & Ownership, Location Details)
- Use proper field types (number, email, textarea)

### 5. List View Update
- Add new columns to display establishment type, contact, etc.

## Implementation Order
1. Create migration for new columns
2. Update model fillable
3. Update controller validation and methods
4. Update form view with new structure
5. Update list view with new columns
