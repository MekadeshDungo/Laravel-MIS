# Plan: Fix Public Announcements Security Issue

## Problem

When a public user visits `/announcements/2` (a single announcement) and clicks "Back to Announcements", they get redirected to an admin-only page because:

1. The public show view (`public-show.blade.php`) links to `/announcements` which uses `announcements.index` controller method
2. The `index()` method in `AnnouncementController` returns `view('announcements.index')` which extends `layouts.admin` (admin-only layout)
3. The admin layout requires `auth()->user()` which causes errors when accessed by public users

This creates a **security issue** - public users are being served admin-only routes that could expose sensitive information or provide unintended access.

## Solution Options

### Option A: Create dedicated public announcements list page (Recommended)
- Create a new public-facing announcements index view using `layouts.client`
- Add a new route `Route::get('/announcements', ...)->name('announcements.public.index')` that uses a new `publicIndex()` method
- Update the public show view to link to this new public route

### Option B: Redirect to home page
- Change "Back to Announcements" link to redirect to home (`/`) instead of announcements list

### Option C: Fix the existing index to handle guests
- Modify `announcements.index` view to work for both authenticated and guest users
- But this is more complex and still mixes public/admin concerns

## Recommended Solution: Option A

### Changes needed:

1. **Create new public index view**: `resources/views/announcements/public-index.blade.php`
   - Use `@extends('layouts.client')`
   - Display published announcements in a public-friendly card layout
   - Include "Read More" links to individual announcement pages

2. **Update route (web.php)**: 
   - The route `/announcements` already points to `index()` which returns `announcements.index`
   - Need to either:
     - Create a new method `publicList()` that returns the new public view
     - Or modify the existing index to detect if user is guest and return appropriate view

3. **Update public show view**:
   - Change "Back to Announcements" link to use the public route

4. **Clean up old views** (optional):
   - Delete or rename `public-index.blade.php` since it's broken (extends admin layout)
   - The current `announcements.index` should remain for admin routes only

## Files to Modify

2. `routes/web.php` - Ensure public route returns public view (may need new controller method)
3. `resources/views/announcements/public-show.blade.php` - Update back link
