# Plan: Fix Registration Form - Fields Not Being Saved to Database

## Problem
The registration form at `/register` is not saving these fields to the database:
- Middle Name
- Block/Lot/Phase/House No.
- Street Name
- Subdivision
- Barangay
- Phone Number
- Date of Birth

City and Province were showing but were disabled inputs that weren't being submitted.

## Root Cause
`RegisteredUserController::store()` only:
1. Validates 4 fields: first_name, last_name, email, password
2. Creates a User record with only: name, email, password, role, is_verified
3. Completely ignores all address and profile fields

## Implementation Plan

### Step 1: Update RegisteredUserController - ALREADY DONE ✓
**File:** `app/Http/Controllers/Auth/RegisteredUserController.php`
- Added validation for all fields
- Creates PetOwner record with all fields after User is created

### Step 2: Update PetOwner Model Fillable - ALREADY DONE ✓
**File:** `app/Models/PetOwner.php`
- Added date_of_birth and email to fillable array

### Step 3: Update Register Views (City/Province) - NEEDS TO DO
**Files:** 
- `resources/views/auth/register.blade.php` (lines 344-356)
- `resources/views/Client/auth/register.blade.php` (lines 344-356)

Current: City and Province are disabled inputs showing "Dasmariñas City" and "Cavite" but they're not being submitted.

Change: Keep the visible disabled inputs (user wants them always visible) and ADD hidden input fields with the same name and value so they get submitted with the form.

Example for City field:
```html
<!-- Keep visible disabled input -->
<div>
    <label class="block text-sm font-medium mb-1.5">City</label>
    <input value="Dasmariñas City" disabled class="w-full px-4 py-2.5 rounded-lg border bg-gray-50 text-gray-500">
</div>
<!-- Add hidden input for form submission -->
<input type="hidden" name="city" value="Dasmariñas City">
```
